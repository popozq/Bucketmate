# 개인화 전략 (Personalization)

> "유저 개개인에 맞춘 AI"로 가는 길. 한 번에 다 만들지 않고 단계로 쌓는다.
> 결정의 "왜"는 [DECISIONS.md](DECISIONS.md), 작업 순서는 [ROADMAP.md](ROADMAP.md).

## 비전
사용자가 고른 전문가 역할(버킷)이 **그 사람을 기억하고**, 대화할수록 더 잘 맞춰주는 개인 AI.

---

## 핵심 구분: 세션 컨텍스트 vs 유저 프로파일
이 둘을 섞으면 설계가 꼬인다. 분리해서 본다.

| | 세션 컨텍스트 | 유저 프로파일 |
|---|---|---|
| 뜻 | "이번 목표/대화"에 대한 정보 | 그 **사람**에 대한 지속적 사실 |
| 예 | "프리랜서 디자이너용 SaaS를 만들려 함" | "디자이너, 비개발자, 주 10h, 한국어 선호" |
| 수명 | 한 세션~한 목표 | 계정이 사는 동안 계속 |
| 저장 | localStorage (Phase 1) | DB (Phase 2~) |

---

## 저장소 현실: localStorage는 어디까지 버티나
- ✅ **브라우저 껐다 켜기 / OS 재부팅 / 같은 브라우저 재방문** → 유지됨 (localStorage는 sessionStorage와 달리 영구)
- ❌ 다른 기기·다른 브라우저 → 없음 (기기마다 따로)
- ❌ 시크릿 모드 종료 / 사용자가 사이트 데이터 삭제 → 삭제
- ⚠️ Safari/iOS(ITP): 7일간 미방문 시 삭제될 수 있음
- → **"같은 브라우저 재방문"까지는 무료로 커버. "어디서나·영구"는 계정이 필요.** 이게 Phase 2의 존재 이유.

---

## 단계 (각 단계가 그 자체로 가치 + 다음 단계로 승격됨)

### ✅ Phase 1 — 대화형 인테이크 (백엔드 0) — 구현 완료 (2026-06-23)
- 빈 텍스트박스 5개(폼) 제거 → 버킷이 **한 번에 하나씩** 묻고 **빠른 답변 칩** 제공(탭 or 직접 입력)
- 컴포넌트: `components/conversational-intake.tsx` (text/single/multi + allowCustom, 진행바, 이전 버튼)
- 데이터 모델: `AgentQuestion`에 `type`/`options`/`allowCustom` 추가. 3개 팩 질문을 선택형 위주로 재설계(한·영).
- 답변은 localStorage(`bucketmate-answers-{locale}`)에 **구조화된 세션 프로파일**로 저장 → 그대로 Phase 2 `user_profile` 스키마 초안.
  - 예: `{ idea, target, stage, skill("기획, 디자인"), time }` (multi는 ", "로 join)
- 검증: 데스크탑·모바일, text/single/multi/custom 전 경로 + 채팅이 답변을 정확히 반영함.
- 효과: 진입 마찰 ↓(타이핑 5회 → 1~2회), 모바일 친화 ↑, 온브랜드("버킷과 대화").

> 🔒 **보안/프라이버시 메모**: Phase 1은 서버 저장이 없어 공격 표면이 작다(데이터는 브라우저 localStorage에만).
> 단, intake 답변은 **AI 컨텍스트로 LLM 제공자(현재 DeepSeek)에 전송**된다 — 이는 프라이버시 고지 대상.
> 민감정보(실명/연락처/결제 등)는 intake에서 받지 않는다. 진짜 보안 작업(암호화·RLS·인증·PII)은 Phase 2(계정/DB)에서.

### Phase 2 — 계정 + 지속 유저 프로파일 (= ROADMAP 1순위 "계정/저장")
- 인증 + DB. 대화형 인테이크가 **유저 프로파일에 기록**됨
- 재방문 시 읽어옴 → "지난번에 ○○ 하고 계셨죠?"
- Phase 1의 세션 프로파일 구조를 거의 그대로 `user_profile` 테이블로

### Phase 3 — 살아있는 프로파일 (진짜 개인 AI)
- 버킷이 대화하며 프로파일을 **스스로 업데이트** (유저 CRM처럼)
- 팩 간 기억 공유: 비즈니스 빌더에서 안 사실을 커리어 헬퍼도 활용

---

## 결정된 방향
- 인테이크는 **선택형/대화형(방향 B)**로 간다. 기존 5개 자유 텍스트 질문은 ChatGPT 플레이스홀더일 뿐, **얽매이지 않음**. 기준은 "AI가 좋은 답을 주려면 뭘 알아야 하나"뿐.
- 데이터 모델: 선택형 하려면 `AgentQuestion`에 `type: 'text'|'select'|'multiselect'` + `options[]` 추가 필요(한·영).
- 헛수고 방지 원칙: **Phase 1의 세션 프로파일 구조 = Phase 2 DB 스키마의 초안.** 처음부터 구조화해 저장.
