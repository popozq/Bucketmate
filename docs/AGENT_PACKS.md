# 에이전트 팩 카탈로그 (Agent Pack Catalog)

> "모든 산업" 타겟을 향한 팩 후보 목록. **한 번에 다 만들지 않는다** — 골라서 깊게.
> 추가 기준(ROADMAP 5번): 사용자 명확 / 반복 업무 / 결과물 구체적 / 규제 위험 낮음 / 워크플로우로 쪼갬 가능.

**우선순위**: `P0` 구현됨 · `P1` 다음 후보 · `P2` 이후

> 🚫 **정책: 규제·자문 분야는 아예 다루지 않는다.** 법률·세무·의료·투자·이민 등은 팩으로 만들지 않는다.
> "정리/체크리스트로만 제한"도 안 함 — **완전히 제외**해 법적 리스크를 원천 차단한다. (DECISIONS.md)
> 이 카탈로그의 모든 팩은 규제 자문이 아닌 **일반 실행·운영·기획** 영역(🟢)이다.

## 추천: 다음에 실제로 만들 P1 (3개)
깊이 우선 + 저위험 + BucketMate로 직접 테스트 가능 기준:
1. **SNS 마케팅 플래너** — 거의 모든 사용자에게 적용, 결과물 구체적, 위험 0
2. **콘텐츠 크리에이터** — 크리에이터 타겟 명확, 반복 업무 많음
3. **학생 학습 플래너** — 사용자 명확, 결과물(공부 계획) 구체적

> 나머지는 사용 데이터/요청 보고 결정. 규제·자문 분야는 후보에서 영구 제외.

## 팩 추가 시 체크리스트 (데이터 모델)
새 팩은 리팩터된 구조를 따른다 (PERSONALIZATION.md):
- 질문: `field` + `scope`(profile/context) + 옵션 `{value, label}` (한·영)
- intake는 자유입력 1~2개 + 선택형 위주 (마찰 ↓)
- `persona` / `systemPrompt` / `suggestedWorkflows` / `accent` / `icon`
- 규제·자문 분야(법률·세무·의료·투자·이민 등)는 팩으로 만들지 않는다.
